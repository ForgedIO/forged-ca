Templates

Reusable certificate templates — allowed EKUs, key usage, validity bounds, name constraints. Bound to ACME provisioners or used for non-ACME CSR signing.

Slice 3 lights this up.
  • CRUD UI for templates — name, key-type allowlist, validity range, EKU set, key usage, SAN name-constraints
  • Default "Web Server (Server + optional Client Auth)" template that ships out-of-the-box
  • ACME provisioners bind to a template (EKU intersection logic: CSR requested EKUs ∩ template-allowed EKUs + forced-always)
  • Non-ACME signing UI picks a compatible template from the list, or admin opts into "Passthrough (advanced)"
  • Federation (slice 13): templates created on Issuing push up to their parent Intermediate; siblings inherit
