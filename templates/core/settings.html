nginx is currently serving the install-time self-signed certificate. This node runs {% if config.is_root and config.is_intermediate %}Root & Intermediate{% elif config.is_root %}Root only{% elif config.is_intermediate %}Intermediate only{% endif %} — neither tier should be signing leaf certificates directly. Keep the self-signed cert until you deploy an Issuing CA.

Why we're holding off.
Standard PKI practice keeps Root and Intermediate keys off the leaf-signing path — their job is to sign other CAs, not web-server certs. Signing this admin UI directly off a Root or Intermediate would use those keys online in a way that defeats the reason you have tiers in the first place.

What to do next

  1. Deploy a second node with the Issuing CA role (next slice wires up federation for this).
  2. Federate it to this node so it gets its Issuing cert signed by your chain.
  3. Come back here and sign the Web UI off the Issuing CA.
