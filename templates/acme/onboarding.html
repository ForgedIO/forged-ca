Client snippets

Pick your client. Each snippet has the directory URL and fingerprint pre-filled — just replace {{ sample_host }} with the DNS name you're enrolling.

{# ── step CLI ── #}

Smallstep's own client. Pins the Root fingerprint out-of-band so you don't have to pre-install the Root.

step ca bootstrap \
  --ca-url {{ ca_url }} \
  --fingerprint {{ root_fingerprint }}

step ca certificate {{ sample_host }} {{ sample_host }}.crt {{ sample_host }}.key \
  --provisioner {{ provisioner.name }} \
  --san {{ sample_host }}
{# ── certbot ── #}

Install the Root in the system trust store first so certbot trusts the CA (instructions here).

sudo certbot certonly \
  --server {{ directory_url }} \
  --standalone \
  -d {{ sample_host }} \
  --email admin@{{ ca_host }} \
  --agree-tos --no-eff-email

# Certs land at /etc/letsencrypt/live/{{ sample_host }}/
{# ── acme.sh ── #}

Shell-only. Good for appliances and lightweight boxes without Python.

export CA_BUNDLE={{ root_download_url }}

acme.sh --issue \
  --server {{ directory_url }} \
  --ca-bundle "$CA_BUNDLE" \
  --standalone \
  -d {{ sample_host }}
{# ── cert-manager (Kubernetes) ── #}

Apply the ClusterIssuer, then on any Ingress annotate cert-manager.io/cluster-issuer: forgedca. Install the Root into the cluster's trust bundle (ConfigMap trusted-ca-bundle) so pods validating the ACME endpoint succeed.

apiVersion: cert-manager.io/v1
kind: ClusterIssuer
metadata:
  name: forgedca
spec:
  acme:
    server: {{ directory_url }}
    email: admin@{{ ca_host }}
    privateKeySecretRef:
      name: forgedca-acme-account
    solvers:
      - http01:
          ingress:
            class: nginx
{# ── Traefik v3 ── #}

Static config. Traefik needs to trust the Root — mount it into the container at /etc/ssl/certs/forgedca-root.crt or put it in the host trust store.

certificatesResolvers:
  forgedca:
    acme:
      email: admin@{{ ca_host }}
      caServer: {{ directory_url }}
      storage: /letsencrypt/acme.json
      httpChallenge:
        entryPoint: web
{# ── Caddy ── #}

Global options in the Caddyfile. acme_ca_root points at the downloaded Root so Caddy trusts the CA directly without depending on system trust.

{
    acme_ca {{ directory_url }}
    acme_ca_root /etc/ssl/certs/forgedca-root.crt
    email admin@{{ ca_host }}
}

{{ sample_host }} {
    reverse_proxy localhost:8080
}
{# ── nginx via certbot --nginx ── #}

certbot's nginx plugin rewrites your vhost in-place. Install the Root in the system trust store first.

sudo certbot --nginx \
  --server {{ directory_url }} \
  -d {{ sample_host }} \
  --email admin@{{ ca_host }} \
  --agree-tos --no-eff-email
{# ── Renewal tips ────────────────────────────────────────────────── #}

Renewal

Default leaves are {% widthratio provisioner.default_leaf_lifetime_hours 24 1 %} days. Good ACME clients auto-renew at ⅓ remaining lifetime, so you almost never touch these commands again. On a Linux host with certbot or acme.sh, the package ships a renew timer; on Kubernetes, cert-manager handles it; on Caddy and Traefik it's built in. step ca renew is one-shot — run it from a cron / systemd timer.
