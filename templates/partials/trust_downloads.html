{# ── ROOT CA — always shown, the primary trust anchor ─────────────── #} {% if config.root_cert_path %}
Root CA certificate Install on endpoints

Install on every device in your fleet that should trust certificates issued here. This is what Windows / Mac / Linux / iOS / Android load to stop getting "untrusted certificate" warnings.

Windows: GPO or Intune macOS: Keychain or MDM Linux: /etc/ca-certificates iOS / Android: MDM profile
root_ca.crt
 {% endif %} {# ── TRUST BUNDLE — only meaningful when an Intermediate exists ───── #} {% if config.is_intermediate %}
Trust bundle Root + Intermediate

The same trust anchors as above, plus the Intermediate CA. Use this instead of Root-only if you want devices to have both trust anchors pre-loaded (smoother if you rotate the Intermediate later).

bundle.crt
 {% endif %} {# ── FULL CHAIN — only when Issuing is present ──────────────────────── #} {% if config.is_issuing %}
Full certificate chain For servers

Root + Intermediate + Issuer concatenated. Serve this alongside your leaf cert on TLS servers — nginx, Apache, Traefik, Caddy, cert-manager — so clients can verify the chain end-to-end.

chain.pem
 {% endif %}

Individual CA certificates (advanced)

Rarely needed separately. Each CA above is also available as its own single-cert file — useful for cert-inspection tooling or building custom trust stores.

{% if config.root_cert_path %} root_ca.crt {% endif %} {% if config.intermediate_cert_path %} intermediate_ca.crt {% endif %} {% if config.issuing_cert_path %} issuer_ca.crt {% endif %}
Automate with curl / Ansible / CI

The trust endpoints are public by default (you can require auth per node if needed) so scripts and fleet-deployment tools can pull them unattended.

# Fetch the chain (example)
curl -ko chain.pem https://{{ config.hostname|default:"localhost" }}:8443{% url 'trust:chain_pem' %}

# Fetch just the Root for fleet trust
curl -ko root_ca.crt https://{{ config.hostname|default:"localhost" }}:8443{% url 'trust:root_crt' %}
