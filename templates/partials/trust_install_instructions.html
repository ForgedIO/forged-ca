Use the bundle.crt download from the Trust page — it concatenates the Root, Intermediate, and Issuing CAs. The OS / browser tooling below imports all three from a single multi-cert PEM and treats each as a trust anchor, which avoids the chain-builder asymmetry where some browsers don't reliably use a wire-supplied Issuing CA.

Windows — system-wide (covers Chrome + Edge)
  1. Double-click the downloaded bundle.crt file.
  2. Click Install Certificate.
  3. Choose Local Machine (requires admin) or Current User, then Next.
  4. Pick Place all certificates in the following store → Browse → Trusted Root Certification Authorities.
  5. Finish. Windows imports each cert in the bundle. Chrome, Edge, and most Windows apps now trust your CA.
macOS — system-wide (covers Safari + Chrome + Edge)
  1. Double-click bundle.crt. Keychain Access opens.
  2. Select the System keychain when prompted.
  3. Three new entries appear — "ForgedCA Root CA", "ForgedCA Intermediate CA", "ForgedCA Issuing CA". For each, double-click, expand Trust, and set When using this certificate to Always Trust.
  4. Close each window (enter your password to save).
Linux — Debian / Ubuntu 
sudo cp bundle.crt /usr/local/share/ca-certificates/forgedca-bundle.crt
sudo update-ca-certificates
Linux — RHEL / Rocky / CentOS Stream 
sudo cp bundle.crt /etc/pki/ca-trust/source/anchors/forgedca-bundle.crt
sudo update-ca-trust
Firefox — maintains its own trust store (do this in addition to the OS step above)
  1. Open about:preferences#privacy (or Settings → Privacy & Security).
  2. Scroll to Certificates and click View Certificates….
  3. On the Authorities tab, click Import… and pick bundle.crt.
  4. Firefox prompts once per cert in the bundle. For each prompt, tick Trust this CA to identify websites and click OK.
  5. Close the certificate manager. Firefox now trusts all three CAs.
